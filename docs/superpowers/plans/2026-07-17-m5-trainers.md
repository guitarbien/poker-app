# M5 訓練模組 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 四個訓練模組（牌型判讀、底池賠率、起手牌範圍、Equity 估算）＋共用測驗迴圈＋錯題本，並把弱點儀表板的「去練習」接上，完成練習迴圈全環。

**Architecture:** 出題/判定/解說是純函式（`src/trainers/`，rng 注入），每題為可序列化物件（錯題本直接存題目 payload）。UI 共用一個 QuizShell（出題→作答→對錯+解說→下一題），四模組只提供題型渲染。進度與錯題本經 storage 模組存 `holdem.trainer.<模組名>`。

**Tech Stack:** 既有，無新依賴。

**Spec:** `docs/superpowers/specs/2026-07-14-holdem-practice-design.md`（§8 訓練模組、§13 M5）

## Global Constraints

- **TDD**：Task 1–2 純邏輯先寫失敗測試；UI（Task 3–4）typecheck + 瀏覽器實測 + e2e
- **每個 commit 前**：`npm run typecheck && npm run test` 全綠；邏輯任務加 coverage（include 加 `'src/trainers/**/*.ts'`）
- **判定重用引擎**：牌型判讀用 `evaluator`、範圍用 `RFI_RANGES/isInRange`、Equity 用 `equity/equityVsRange`——**不寫第二份判定邏輯**
- **錯題本規則（spec §8）**：答錯自動收入；「複習錯題」入口僅從錯題本抽題；同一題**連續**答對兩次（任何模式，中間答錯歸零）移出
- **Equity 估算計分（spec §8.4）**：誤差 ≤5 個百分點算答對，只計入該模組正確率
- 不新增依賴；介面繁體中文；commit 繁體中文；POSIX；e2e 埠 5199

```ts
// handReading.ts —— 兩種題型隨機出
export type HandReadingPayload =
  | { kind: 'best'; cards: Card[]; options: string[]; answerIdx: number }      // 7 張，四選一牌型名
  | { kind: 'compare'; holeA: [Card, Card]; holeB: [Card, Card]; board: Card[]; answer: 'A' | 'B' | 'tie' };
export function generateHandReading(rng: Rng): QuizQuestion<HandReadingPayload>;
export function judgeCompare(holeA: [Card, Card], holeB: [Card, Card], board: Card[]): 'A' | 'B' | 'tie';  // 匯出供測試 rigged tie 案例（隨機 tie 率僅 ~4%，不可靠隨機掃）
export function explainHandReading(p: HandReadingPayload): string;  // 用 bestHand/describeHand（自 engine/handNames）組解說（含組成五張）

// potOdds.ts —— 四選一所需勝率區間
export interface PotOddsPayload {
  pot: number; call: number;
  options: string[];        // 如 ['約 15%', '約 25%', '約 33%', '約 50%']（正解 + 三個干擾）
  answerIdx: number;
  required: number;         // 精確值，解說用
}
export function generatePotOdds(rng: Rng): QuizQuestion<PotOddsPayload>;
export function explainPotOdds(p: PotOddsPayload): string;  // 「跟注 C ÷（底池 P + 跟注 C）= X%」

// preflopRange.ts —— 開牌/棄牌二選一
export interface PreflopRangePayload {
  position: Exclude<Position, 'BB'>;
  hole: [Card, Card];
  answer: 'open' | 'fold';   // isInRange(RFI_RANGES[position])
}
export function generatePreflopRange(rng: Rng): QuizQuestion<PreflopRangePayload>;
// 13×13 矩陣資料（UI 渲染用）：169 格，各格 {label, inRange, isCurrent}
export function rangeMatrix(position: Exclude<Position, 'BB'>, hole: [Card, Card]): { label: string; inRange: boolean; isCurrent: boolean }[][];

// equityGuess.ts —— 輸入百分比估勝率
export interface EquityGuessPayload {
  hole: [Card, Card];
  board: Card[];             // 0/3/4 張（preflop/flop/turn）
  opponent: { kind: 'hand'; hole: [Card, Card] } | { kind: 'range'; name: string; range: string };
  // range 用預設清單（名稱→範圍字串）：如 '緊手開牌' = '77+, ATs+, KQs, AJo+'、'寬鬆跟注' = '22+, A2s+, K9s+, QTs+, JTs, ATo+, KJo+'
}
export function generateEquityGuess(rng: Rng, presets?: RangePreset[]): QuizQuestion<EquityGuessPayload>;  // presets 可注入（測試 fallback 用）
export function rangeHasLiveCombo(range: Range, dead: Card[]): boolean;  // 防禦檢查（預設 presets 下數學上不可能 block——'77+' 需 3 張同 rank 死牌才能 block 一個對子類別；此路徑僅注入自訂 presets 可測）
// 判定：回傳實際 equity 與是否答對（|guess*100 − actual*100| ≤ 5）
// 注意：對「明牌對手」engine 沒有現成函式（equity() 是隨機對手）——在本檔自寫 equityVsHand 小型 MC（~20 行，重用 evaluate7）
export function judgeEquityGuess(p: EquityGuessPayload, guessPercent: number, rng: Rng, iterations?: number): { actual: number; correct: boolean };
```

出題品質要求（測試錨點；數值規則以下為定案）：
- handReading `best`：四個選項為不同牌型名、正解 = `decodeScore(best7(cards).score).category` 對應名；干擾項從鄰近類別挑（±1、±2），不足時往外擴
- potOdds：pot ∈ {20,30,...,200}、call ∈ pot 的 {1/4,1/3,1/2,2/3,1}（**Math.round 取整**）；正解顯示為「約 X%」點值（Math.round，實際值域 20–50%）——**刻意以點值取代 spec 的「區間」措辭**（間距 ≥5pp 即隱含區間語意）；干擾項從步距 5 的網格 **{10,15,…,65}** 取、先濾掉與正解差 <5 的格點（濾後恆有 ≥3 個可選）
- preflopRange：position 均勻抽（UTG/MP/CO/BTN/SB）；手牌完全隨機兩張
- equityGuess：opponent 為明牌或範圍各 50%；board street 均勻；出題時以 `rangeHasLiveCombo` 防禦檢查、失敗改出明牌題（預設 presets 下不會發生）
- **id 決定性**：id 只編碼「排序後的卡牌與題目參數」，**不含 options 順序**（錯題本比對語意：同一題不同選項排列視為同題；payload 本身存了 options 所以重出題無虞）
- rangeMatrix 慣例（UI 對齊用）：rank 遞減（A→2），對角線＝對子，上三角＝suited（列 rank 較高），下三角＝offsuit（'AKs' 在 [0][1]、'AKo' 在 [1][0]、'72o' 在 [12][7]）

- QuizShell 共用迴圈：顯示題目 → 作答 → 立即對錯（✓/✗ 配色）＋解說 → 「下一題」；右上顯示本次連對/答題數；作答後即 `recordAnswer` + `saveProgress`
- 題型渲染：
  - 牌型判讀：7 張 CardView + 四鈕；compare 題型兩組手牌＋共用 board、三鈕（A/B/平手）；解說顯示組成五張（`bestHand().five` 高亮）與 `describeHand`

  - 起手牌範圍：位置徽章＋兩張牌＋「開牌/棄牌」二鈕；解說渲染 `RangeMatrix`（13×13，inRange 上色、isCurrent 外框）
  - Equity 估算：手牌＋board＋對手（明牌顯示兩張；範圍顯示名稱＋範圍字串）＋number input（0–100）＋「作答」；解說顯示實際勝率與誤差。判定為同步呼叫（數十毫秒可接受，**不做「計算中」文案**——同步阻塞下 loading 永遠 paint 不出來）；**判定一次後保留 actual**（同題重判 MC 會漂移 ±1pp，±5 邊界附近會翻轉）
- 複習模式：**依 wrongBook 陣列順序輪詢（FIFO）**，題目移出即從清單消失，清單空顯示完成文案返回
